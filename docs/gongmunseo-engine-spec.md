## 0. 현재 generator의 실측 단위 규약 (구현 기준)

`generator.ts` 코드에서 직접 확인한 단위 매핑 — 새 모드는 이 규약을 그대로 따른다.

| 개념 | 함수/속성 | 단위 규약 | 실측 예 (현재 코드) |
|------|-----------|-----------|---------------------|
| 글자 크기 | `charPr(id, height, …)` → `<hh:charPr height="…">` | **height = pt × 100** | 1000=10pt, 900=9pt, 1800=18pt(h1), 1400=14pt(h2), 1200=12pt(h3), 1100=11pt(h4) |
| 줄간격 | `paraPr(id, {lineSpacing})` → `<hh:lineSpacing type="PERCENT" value="…">` | **퍼센트 정수** | 160(기본), 180(h1), 130(코드) |
| 들여쓰기(좌여백/내어쓰기) | `paraPr(id, {indent})` → `<hh:margin indent="…">` | **HWPUNIT** | 400(코드블록), 600(인용/리스트) |
| 문단 위/아래 간격 | `{spaceBefore, spaceAfter}` → `<hh:margin prev/next>` | **HWPUNIT** | 800/200(h1) |
| 정렬 | `{align}` → `<hh:align horizontal="…">` | `LEFT/CENTER/RIGHT/JUSTIFY` | 기본 JUSTIFY |
| 용지·여백 | `generateSecPr()` → `<hp:pagePr><hp:margin>` | **HWPUNIT, 1mm ≈ 283.46 HWPUNIT** | width=59528(210mm), height=84188(297mm) |

**HWPUNIT 환산 상수**: `1mm = 7200/25.4 ≈ 283.465 HWPUNIT`. `1pt = 100 (charPr height)`. **1타(영문·숫자 1자 폭) ≈ 본문 글자폭의 절반 = 약 0.5em.** 12pt 본문 기준 1em ≈ 12pt ≈ 1200 charPr-height이나, **들여쓰기(paraPr indent)는 폭 HWPUNIT** 이므로 별도 환산 필요(아래 2장).

> ⚠️ **현재 generator의 secPr 여백은 비표준이다.** 코드 주석/값: 위 30mm(8504)/아래 15mm(4252)/좌 20mm(5670)/우 15mm(4252). REFERENCE 검증 결과 공식값은 **위 20 / 아래 10 / 좌 20 / 우 20mm**. 공문서 모드에서는 이 값을 교체해야 한다(3.3 참조).

## (a) 마크다운에서 항목부호 8단계 감지/표현

### a-1. 입력 표현 — 마크다운 중첩 리스트로 매핑

현재 파서(`parseMarkdownToBlocks`)는 리스트를 `indent = floor(선행공백/2)`로 잡고 `ordered`(숫자) / 비순서(`-*+`)만 구분한다(generator.ts L207-213). 공문서 8단계는 **들여쓰기 깊이(0~7)** 로 1:1 대응시키되, 부호 종류는 **깊이별 고정 매핑**으로 강제한다(마크다운의 `1.`/`-` 종류 무시).

| MD 들여쓰기 depth | 공문서 단계 | 출력 부호 | 카운터 진행 |
|-------------------|------------|-----------|-------------|
| 0 | 1 첫째 | `1.` `2.` … | 아라비아 |
| 1 | 2 둘째 | `가.` `나.` … | 가나다 → (소진 시) 거/너/더 |
| 2 | 3 셋째 | `1)` `2)` … | 아라비아 |
| 3 | 4 넷째 | `가)` `나)` … | 가나다 |
| 4 | 5 다섯째 | `(1)` `(2)` … | 아라비아 (괄호 3글자) |
| 5 | 6 여섯째 | `(가)` `(나)` … | 가나다 (괄호 3글자) |
| 6 | 7 일곱째 | `①` `②` … | 원숫자 U+2460+ |
| 7 | 8 여덟째 | `㉮` `㉯` … | 원한글 U+326E+ |

### a-2. 부호 생성 규칙

- **레벨별 독립 카운터** + **상위 번호 변경 시 하위 리셋**(현재 `orderedCounters`가 이미 indent별 카운터 + 하위 리셋 로직 보유, L580-639 — 재사용).
- 가나다 시퀀스: `['가'..'하']`(14자) 소진 후 `거/너/더…` 단모음 연속(REFERENCE 1.5). 구현: 초성 인덱스 배열 + 중성 순환.
- 원숫자: `String.fromCodePoint(0x2460 + n)` (①~⑳ = U+2460~U+2473). 원한글: `String.fromCodePoint(0x326E + n)` (㉮~㉻).
- **5·6단계는 반드시 괄호 3글자 문자열**(`(`+숫자/한글+`)`), 단일 유니코드 ⑴/㈎ 금지.
- **항목이 1개뿐인 레벨은 부호 미부여**(REFERENCE 1.2) — 같은 depth·같은 부모에서 형제 수=1이면 부호 없이 내용만. (2-pass: 1차에서 형제 수 집계, 2차에서 부호 결정.)

### a-3. 부호와 텍스트 분리(권장 구현)

현재 generator는 리스트를 `indentPrefix + marker + text`로 **하나의 run**에 합쳐 공백 들여쓰기로 표현(L636-637). 공문서 모드는 **공백 들여쓰기 대신 paraPr indent + 내어쓰기**를 써야 줄바꿈 정렬(REFERENCE 1.3)이 깨지지 않는다. 따라서:
- 텍스트 = `부호 + 1타공백 + 내용` (한 run), 들여쓰기는 paraPr로(아래 b장).
- 1타 공백은 부호 뒤 실제 공백 1개(`가. 내용`)로 출력.

## (b) 들여쓰기·내어쓰기·줄간격 → paraPr 매핑

### b-1. 1타 → HWPUNIT 환산

REFERENCE: **2타 = 한글 1자 = 영문·숫자 2자**, **1타 = 영문·숫자 1자 폭 ≈ 0.5 한글자**. 본문 15pt 기준:
- 한글 1자 폭 ≈ 1em ≈ 15pt ≈ **약 1417 HWPUNIT** (15pt × 283.465/72 × ... → 실용 근사 `1자폭 ≈ height/100 × 283.465/2.835`). 실무 근사: **1타 ≈ 한글 글자폭의 1/2 ≈ 700~750 HWPUNIT(15pt)**, **2타 ≈ 1400~1500 HWPUNIT**.
- 단순·안정 구현 권장값: **2타 = 1400 HWPUNIT**(≈한글 1자, 15pt 기준), **1타 = 700 HWPUNIT**. (글자크기 가변 시 `2타 = round(height × 1.4)` 식으로 동적 산출 가능하나, 한글 표준 실무는 고정 폭 들여쓰기로 충분.)

### b-2. 단계별 paraPr (공문서 본문 전용 paraPr 신설)

각 단계마다 **좌여백(left) = 누적 2타** + **내어쓰기(indent 음수) = 부호폭+1타** 를 둔다. `<hh:margin left="…" indent="…">` — `left`는 문단 전체 좌여백, `indent`는 첫 줄 추가 들여쓰기(음수면 내어쓰기/hanging).

> 현재 `paraPr()`는 `indent`만 받고 `left=0` 고정(L378). 공문서 모드는 `paraPr` 헬퍼에 `left` 파라미터를 추가해야 한다.

| 단계 | 부호 | 누적 들여(타) | left (HWPUNIT) | 부호폭+1타 (hanging indent, 음수) | lineSpacing |
|------|------|--------------|----------------|-----------------------------------|-------------|
| 1 | 1. | 0 | 0 | -2100 (부호 2타+1타 ≈ 3타) | 공문서 기본(아래) |
| 2 | 가. | 2 | 1400 | -2100 | 〃 |
| 3 | 1) | 4 | 2800 | -2100 | 〃 |
| 4 | 가) | 6 | 4200 | -2100 | 〃 |
| 5 | (1) | 8 | 5600 | -2800 (3글자 부호 → 4타) | 〃 |
| 6 | (가) | 10 | 7000 | -2800 | 〃 |
| 7 | ① | 12 | 8400 | -2100 | 〃 |
| 8 | ㉮ | 14 | 9800 | -2100 | 〃 |

- **hanging indent**: `<hh:margin left="L" indent="-H">` → 첫 줄은 `L-H`(부호가 더 왼쪽), 둘째 줄부터 `L`(내용 첫 글자에 맞춰 정렬). 이것이 REFERENCE 1.3 "둘째 줄 내용 첫 글자 정렬"을 paraPr로 구현하는 방법.
- 단순화 옵션(내어쓰기 생략 가능): "왼쪽 기본선 시작도 무방"(REFERENCE 1.3)이므로, indent 없이 `left=누적2타`만 줘도 규정 위반 아님. **권장은 hanging**(가독성).

### b-3. 줄간격 — 공문서 모드 기본값

REFERENCE 2.3: 단일 표준값 없음 → **프리셋으로 분기**(아래 e장). 공문서 모드 기본 본문 줄간격:
- **기안문/통지**: 160% (보고서 관행·가독성). 또는 시스템 호환 위해 130%.
- **큰글자 서식 준용**: 130% 이상(법정 하한).
- 현재 `paraPr` 기본 lineSpacing=160(L372) — 기안문엔 그대로 두되 프리셋에서 override.

> 코드에 단일값 하드코딩 금지(검증 confirmed). 프리셋 파라미터로 노출.

### e-1. 보고서 vs 기안문 항목부호 차이 (핵심)
- **기안문/통지**: 시행규칙 8단계(`1. 가. 1) 가) (1) (가) ① ㉮`) 적용.
- **보고서**: 최상위 로마숫자 `Ⅰ.Ⅱ.Ⅲ` + 항목 `□`(대)`○`(중)`-`(소)`·`(세) — 관행. depth→부호 매핑을 프리셋별로 분기.

### e-2. secPr 여백 교체 (공통 — 현재 비표준값 수정)
```
현재(generator.ts L505): top=8504(30mm) bottom=4252(15mm) left=5670(20mm) right=4252(15mm)
공문서 표준으로 교체:    top=5670(20mm) bottom=2835(10mm) left=5670(20mm) right=5670(20mm)
머리말/꼬리말: header=0 footer=0  (현재 2835 → 0)
환산: 20mm=5669.3→5670, 10mm=2834.6→2835. (1mm=283.465 HWPUNIT)
```

## 구현 체크리스트 (generator.ts 변경점)

1. `paraPr()` 헬퍼에 `left` 파라미터 추가(현재 left=0 고정) — b-2 hanging indent용.

## ⚠️ 구현 반영 정정 (2026-06-19, 실제 한컴 공문서 파일 검증)

위 (b) 장의 `left`/`indent` 산식은 **HWPX 출력 형식과 한컴 렌더 의미를 실측으로 정정**했다. 실제 한컴이 만든 공문서 HWPX(`「아차산메아리」 게재 요청의 건.hwpx` 등)를 디코드해 확인한 결과:

1. **`<hh:margin>`은 속성형이 아니라 자식요소형이다.** 한컴은 `<hh:margin indent="…" left="…">`(속성)를 **무시**하고, 다음 자식요소만 읽는다 (`xmlns:hc="http://www.hancom.co.kr/hwpml/2011/core"` 선언 필수):
   ```xml
   <hh:margin>
     <hc:intent value="-2250" unit="HWPUNIT"/>   <!-- 첫 줄 들여쓰기, 음수=내어쓰기 -->
     <hc:left value="1500" unit="HWPUNIT"/>        <!-- 왼쪽 여백(단계 누적) -->
     <hc:right value="0" unit="HWPUNIT"/>
     <hc:prev value="0" unit="HWPUNIT"/>           <!-- 문단 위 간격 -->
     <hc:next value="0" unit="HWPUNIT"/>           <!-- 문단 아래 간격 -->
   </hh:margin>
   ```
2. **내어쓰기 의미 정정**: `<hc:intent>` **음수** = 첫 줄은 `left`에서 시작하고 **둘째 줄부터 `|intent|` 만큼 오른쪽으로** 들여쓴다(내용 첫 글자 정렬). (이전 스펙의 "first line = left+indent" 모델은 오해였음.)
3. **올바른 산식** (`levelIndent`): `left = depth × 2타`(누적), `intent = -(부호폭 + 1타)`. 1타 = bodyHeight/2 HWPUNIT.
   - 예(15pt, bodyHeight=1500): depth0 `left=0 intent=-2250`, depth1 `left=1500 intent=-2250`, depth2 `left=3000 …` — 실파일의 `left` 누적 + 음수 `intent` 패턴과 동일.
4. 용지 여백(`<hp:pagePr><hp:margin>`)은 **속성형이 맞다**(실파일도 속성형). 실파일 실측 = `left=5669 right=5669 top=5669 bottom=2835 header=0 footer=0` = 위20/좌우20/아래10·머리말꼬리말0 → 본 스펙 공식값과 일치.
5. 본문 글꼴 크기: 실측 공문(광진구)은 height=1000(10pt). 보고서 관행(14~15pt)과 별개로 일반 공문 본문은 더 작은 경우가 많음 → `bodyPt` 옵션으로 조정 가능.

> 교훈: HWPX는 **XML 유효성·라운드트립 통과만으로 렌더 정확성을 보장하지 못한다.** 서식(여백·들여쓰기)은 반드시 **한글/한컴으로 실제 렌더를 확인**하거나, 실제 한컴 산출 파일과 구조를 대조할 것.

- **h1(첫 번째)** → 표지(파랑 장식 바 + 제목 + 날짜 + 기관명). `cover: false`로 끄거나 `{date, org}` 지정. 표지 소비 후 나머지 h1은 장 헤더로.

- **리스트 depth** → □ ○ ― ㆍ (시프트 반영). **`※`로 시작하는 문단/항목·blockquote** → ※ 참고 스타일.

### 줄나눔 — `breakNonLatinWord` 이름 역전 (2026-07-11 실측 확정)

한컴 실구현에서 이 속성은 **이름과 반대로** 동작한다. 어절 단위를 원하면 `BREAK_WORD`를 써야 한다.

| 속성 | `KEEP_WORD` | `BREAK_WORD` |
|---|---|---|
| `breakLatinWord` (영어) | 단어 유지 — 이름대로 | 글자 분리 |
| `breakNonLatinWord` (한글) | **글자 분리** | **어절 유지** |

- 근거: 한글 COM(SaveAs PDF) 실렌더 A/B 매트릭스 — 동일 문서에서 속성 하나씩 토글.
  `KEEP_WORD`는 정렬(JUSTIFY/LEFT) 무관 "실증되었/고," 식 어절 중간 분리,
  `BREAK_WORD`는 전 줄바꿈 어절 경계 + 영어 단어 통째 유지. `snapToGrid`·`condense`는 줄나눔 방식과 무관.
- 정본 양식(2_보고서)의 본문이 `KEEP_WORD/KEEP_WORD`인 것은 한글 **기본값**(한글단위 '글자'/영어단위 '단어') 그대로 저장된 것.
- 산출 정책: 공문서·일반 모든 경로 `breakNonLatinWord="BREAK_WORD"`(어절)·`breakLatinWord="KEEP_WORD"`(단어). 줄폭보다 긴 단일 토큰(긴 URL·무공백 `·` 연쇄)만 강제 분해(한컴 동일).
